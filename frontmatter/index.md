# Concepts

* [Aliases](aliases.md) - How the aliases field gives a note extra names for name-based links, matched at the lowest resolver tier.
* [Categories](category.md) - How the category field is used to classify concepts in this wiki.
* [Id](id.md) - How the id field gives a note a stable identifier that survives renames and moves.
* [LLM](llm.md) - How the llm field records an LLM's relationship to a note — authorship, review/trust status, and whether agents may edit it.
* [Resource](resource.md) - How the resource field records canonical URI(s) for the underlying asset a concept documents.
* [Description](description.md) - How the description field records a one-line summary of a concept.
* [See also](see-also.md) - How the see-also field records related notes as a machine-readable relation, complementing inline body links.
* [Sensitivity](sensitivity.md) - How the sensitivity field classifies how private or protected a note's content is — required on person and personal-data dataset notes.
* [Superseded by](superseded_by.md) - How the superseded_by field marks a note as replaced by a newer one and points to its successor.
* [Title](title.md) - How the title field records a concept's display name.
* [Type](type.md) - How the type field marks a concept's structural role in the wiki — the one required frontmatter field.
* [Visibility](visibility.md) - How the visibility field controls whether a note is indexed, searched, and surfaced by agents.
* [Tags](tags.md) - How the tags field is used for free-form keywords and cross-cutting association in this wiki.
